# JVM系列：编译

程序代码需要编译后才能在虚拟机中运行，前端编译、即时编译（JIT编译）、静态提前编译（AOT编译）

# 前端编译器

把Java源码文件（.java）编译成class文件(.class)的过程，也就是javac编译器。

# 流程

# 词法分析器

说明

主要是读取源代码，一个字节为一节地读取进来，那些是语言的关键字，例如，if、else等， 并从源代码中找出一些规范化的Token流，一个形象的比如就是人类能分清楚一个句子的单词和标点符号。

  • 语法分析器

说明

就像人类能分清楚一个句子的主语谓语宾语，检查是否符合语法。语法分析的结果就是形成一个Java语言规范的抽象语法树。

  • 语义分析器

说明

将一些复杂难懂的语法转化为简单易懂的语法，例如给类添加默认的构造器函数，检查变量在使用前是否已经初始化，将一些常量进行合并处理，检查操作变量类型是否匹配，检查所有的操作数据是否可达，检查checked exception异常是否已经捕获或者破除，解除Java语法糖等

# JIT编译器

通过Java虚拟机（JVM）内置的即时编译器（Just In Time Compiler，JIT编译器）；在运行时把Class文件字节码编译成本地机器码的过程
